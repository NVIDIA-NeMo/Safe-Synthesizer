# Privacy

NeMo Safe Synthesizer provides multiple layers of privacy protection for synthetic data generation.

## PII Replacement

The PII replacer detects personally identifiable information using Named Entity Recognition (NER) models and regex patterns, then replaces detected entities with synthetic but realistic values.

### Supported PII Types

- Names (person, organization)
- Email addresses
- Phone numbers
- Social Security Numbers
- Physical addresses
- Dates of birth
- Custom patterns via regex

### How It Works

1. **Detection**: NER models and regex patterns scan each record for PII entities
2. **Replacement**: Detected entities are replaced with synthetic values generated by [Faker](https://faker.readthedocs.io/)
3. **Consistency**: Column statistics and distributions are preserved

## Differential Privacy

NeMo Safe Synthesizer supports Differential Privacy (DP) training via [Opacus](https://opacus.ai/) integration.

### Key Parameters

| Parameter | Description |
|-----------|-------------|
| `epsilon` | Privacy budget -- lower values mean stronger privacy |
| `delta` | Probability of privacy guarantee failure |
| `max_grad_norm` | Maximum gradient clipping norm |

### How It Works

DP-SGD (Differentially Private Stochastic Gradient Descent) adds calibrated noise to gradients during training, providing a mathematical privacy guarantee that limits what can be learned about any individual record.

## Privacy Evaluation

After generation, the evaluation system measures privacy through:

- **PII Replay Detection** -- checks if original PII values leak into synthetic data
- **Membership Inference Protection** -- measures resistance to MI attacks
- **Attribute Inference Protection** -- measures resistance to AI attacks

See [Evaluation](evaluation.md) for details on all privacy metrics.
